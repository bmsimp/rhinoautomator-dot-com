ADMITTING DEFEAT
ON THE DATABASE

Yeah, ok. This is where I had to admit that I was going to need some kind of "database". I still cheated though. Since most systems shouldn't have incidents/maintenance that often, I decided to use a script template in Rewst. The script template is set up in YAML and then immediately converted to JSON using the |from_yaml_string filter. If you've never done this before, you can set a data alias under a noop transition at the start of your workflow and use this format:

{{ template("template GUID copied from end of URL for template")|from_yaml_string }}
